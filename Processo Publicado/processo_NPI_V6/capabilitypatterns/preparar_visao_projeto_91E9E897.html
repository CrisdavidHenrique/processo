Task Descriptor: Preparar Visão do Projeto
Esta tarefa consiste na identificação e especificação de maneira formal dos requisitos do software a ser construído.
Based on Method Task: Preparar Visão do Projeto
Relationships
RolesPrimary:

  • Professor Supervisor
Assisting:
Outputs
  • Documento de Especificação dos Requisitos
  • Documento de Visão
Steps
1) Identificar os requisitos
O lider técnico, juntamente com a equipe do projeto deverá elaborar o documento de visão do projeto, a partir das informações sobre o projeto fornecidas pelo professor supervisor, porém estas informações podem não ser suficientes para a elaboração do documento. Portanto, a equipe poderá realizar reuniões com o cliente e através de entrevistas, observação direta ou outras técnicas de elicitação, será feito o levantamento das necessidades do sistema. Essas necessidades serão documentadas através de texto escrito ou de gravações das entrevistas. As informações adquiridas servirão de base para a elaboração do documento de visão.
2) Especificar os requisitos

O documento de visão deve ser refinado para conter os requisitos funcionais e não funcionais do sistema.
